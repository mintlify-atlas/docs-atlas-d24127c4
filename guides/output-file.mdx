title: "Output file"

After every generation cycle, A-Maze-ing writes the current maze state to a plain-text file. The destination is set by the `OUTPUT_FILE` key in your config (default: `maze.txt`).

<Note>
  The output file is **overwritten** on every generation cycle. If you need to keep a particular maze, copy the file before pressing [1] to regenerate.
</Note>

## File structure

`generate_output()` in `output.py` writes four sections in order:

1. The maze grid encoded as hexadecimal characters, one row per line
2. A blank line
3. Entry coordinates: `x,y`
4. Exit coordinates: `x,y`
5. The solution path as a string of direction characters

```python
with open(output_data['OUTPUT_FILE'], 'w', encoding="utf-8") as output:
    output.write(maze_txt)          # hex grid rows
    output.write("\n")              # blank separator
    output.write(f"{output_data['ENTRY'][0]},{output_data['ENTRY'][1]}\n")
    output.write(f"{output_data['EXIT'][0]},{output_data['EXIT'][1]}\n")
    output.write(f"{output_data['PATH']}\n")
```

## Hex grid encoding

Each cell is stored as a single uppercase hexadecimal character representing which walls are present. `int_to_hex()` converts the integer grid:

```python
def int_to_hex(maze: List[List[int]]) -> List[List[str]]:
    grid: List[List] = []
    for line in maze:
        row = ""
        for cell in line:
            row += str(hex(cell)[2:]).upper()
        grid.append(row)
    return grid
```

The cell integer is a 4-bit wall bitfield (North=1, East=2, South=4, West=8), so valid values are `0`–`15`, mapping to hex characters `0`–`F`:

| Hex | Binary | Walls present           |
|-----|--------|-------------------------|
| `0` | `0000` | None (all open)         |
| `6` | `0110` | East + South            |
| `F` | `1111` | All four walls closed   |

## Direction encoding

`dir_to_path()` converts the list of solution coordinates into a compact direction string. Each step compares two consecutive positions and maps the delta to a letter:

```python
def dir_to_path(path: List[tuple]) -> str:
    moves = {(0, -1): "N",
             (1,  0): "E",
             (0,  1): "S",
             (-1, 0): "W"}

    for (cx, cy), (nx, ny) in zip(path, path[1:]):
        direction = (nx - cx, ny - cy)
        route.append(moves.get(direction, "X"))
    return "".join(route)
```

## Example output

For a 20×15 maze with entry `(0,0)` and exit `(19,14)` (the defaults from `config.txt`):

```text maze.txt
FFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFF
F7FFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFF9F
...

0,0
19,14
EESSSEEENNNNNNEEESSSSEENNSSEE...
```

- Every `F` on the border rows represents a fully-walled cell (`1111`) — the outer boundary before doors are opened.
- The blank line separates the grid from the coordinate/path metadata.
- The path string uses only `N`, `E`, `S`, `W` characters, one per step from entry to exit.
