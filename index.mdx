## What is A-Maze-ing?

**A-Maze-ing** is a Python CLI tool that generates and solves mazes interactively in your terminal. It uses a depth-first search (DFS) algorithm to carve perfect mazes and a breadth-first search (BFS) algorithm to find the shortest path between any entry and exit point.

Run it, watch your maze come to life, toggle the solution path, change colors, and regenerate — all from a simple interactive menu.

```
=== A-Maze-ing ===
[1]. Re-generate a new maze
[2]. Show path from entry to exit
[3]. Change maze colors
[4]. Quit
```

## Key features

<CardGroup cols={2}>
  <Card title="DFS Maze Generation" icon="shuffle">
    Generates perfect mazes using randomized depth-first search with backtracking — every cell is reachable from every other cell.
  </Card>
  <Card title="BFS Pathfinding" icon="route">
    Finds the shortest path from entry to exit using breadth-first search, displayed as a highlighted trail through the maze.
  </Card>
  <Card title="Animated Rendering" icon="play">
    Watch the maze being carved and the solver exploring paths in real time, with configurable animation speed.
  </Card>
  <Card title="Color Themes" icon="palette">
    Cycle through CYAN, BLUE, PURPLE, and YELLOW wall color themes without regenerating the maze.
  </Card>
  <Card title="Non-Perfect Mode" icon="arrows-split-up-and-left">
    Optional imperfect maze mode that removes extra walls to create multiple valid paths through the maze.
  </Card>
  <Card title="The 42 Pattern" icon="star">
    Mazes large enough (9×7 or bigger) embed a hidden "42" pattern in the center as a protected island of walls.
  </Card>
  <Card title="File Output" icon="file-export">
    Exports the maze grid, entry/exit coordinates, and the solution path to a `.txt` file in hexadecimal format.
  </Card>
  <Card title="Reproducible Seeds" icon="seedling">
    Optionally seed the random number generator to reproduce the exact same maze layout every time.
  </Card>
</CardGroup>

## How it works

A-Maze-ing is built around the `mazegen` Python package, which provides the maze generation, solving, and rendering engine. The CLI entry point `a_maze_ing.py` reads a configuration file, builds the maze, and drops you into an interactive session.

<Steps>
  <Step title="Write a config file">
    Define your maze dimensions, entry/exit coordinates, and optional settings in a `config.txt` file.
  </Step>
  <Step title="Run the program">
    Execute `python3 a_maze_ing.py config.txt` — the maze generates and renders immediately.
  </Step>
  <Step title="Interact">
    Use the numbered menu to regenerate, show/hide the solution, cycle colors, or quit.
  </Step>
</Steps>
