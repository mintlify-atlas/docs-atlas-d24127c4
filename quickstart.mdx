Get A-Maze-ing running and generate your first maze in just a few steps.

<Steps>
  <Step title="Clone and install">
    Clone the repository and install dependencies:

    ```bash
    git clone https://github.com/carlosgilnieto/a-maze-ing.git
    cd a-maze-ing
    make install
    ```

    This installs the `mazegen` package and any dependencies. See [Installation](/installation) for full details.
  </Step>

  <Step title="Create a config file">
    Create a `config.txt` file (or use the one included in the repo) that defines your maze settings:

    ```text config.txt
    WIDTH=20
    HEIGHT=15
    ENTRY=0, 0
    EXIT=19,14
    OUTPUT_FILE=maze.txt
    PERFECT=True
    #SEED=42 # Optional
    #ANIMATION=True # Optional but need SPEED_ANIMATION
    #SPEED_ANIMATION=0.01 # Optional but need ANIMATION
    ```

    This creates a 20×15 maze with the entry at the top-left corner `(0,0)` and the exit at the bottom-right `(19,14)`. Output is written to `maze.txt`.

    | Key | Value | Description |
    |-----|-------|-------------|
    | `WIDTH` | `20` | Number of columns |
    | `HEIGHT` | `15` | Number of rows |
    | `ENTRY` | `0, 0` | Entry point (x, y) |
    | `EXIT` | `19, 14` | Exit point (x, y) |
    | `OUTPUT_FILE` | `maze.txt` | Output filename |
    | `PERFECT` | `True` | Generate a perfect maze |
  </Step>

  <Step title="Run the program">
    Launch A-Maze-ing with your config file:

    <CodeGroup>

    ```bash Python
    python3 a_maze_ing.py config.txt
    ```

    ```bash Make
    make run
    ```

    </CodeGroup>

    The maze generates instantly and renders in your terminal using ASCII box-drawing characters and ANSI colors.
  </Step>

  <Step title="Use the interactive menu">
    After the maze renders, you see the interactive menu:

    ```text
    === A-Maze-ing ===
    [1]. Re-generate a new maze
    [2]. Show path from entry to exit
    [3]. Change maze colors
    [4]. Quit

    Choice? (1-4):
    ```

    | Option | Action |
    |--------|--------|
    | `1` | Regenerate a brand-new maze using DFS |
    | `2` | Toggle the BFS shortest-path overlay on/off |
    | `3` | Cycle to the next wall color (CYAN → BLUE → PURPLE → YELLOW) |
    | `4` | Clear the terminal and quit |

    <Tip>
      Press `2` to reveal the shortest path from your entry point (green) to the exit (red). Press it again to hide it.
    </Tip>
  </Step>
</Steps>

<Tip>
  If your maze is at least 9 columns wide and 7 rows tall, look for the "42" pattern embedded in the center — it's an easter egg formed by protected wall cells that the DFS algorithm never carves through. See [The 42 pattern](/advanced/pattern-42) for details.
</Tip>
